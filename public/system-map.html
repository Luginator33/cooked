The problem: Your Profile page and other people viewing your profile read from user_data.loved in Supabase. But the love button never writes there directly. The debounced auto-save writes user_data.heat.loved (nested inside the heat object), but the Profile reads the top-level loved array.

6. What To Fix (Priority Order)

High Priority
  1. Unify love system — One source of truth. toggleLove should update user_data.loved in Supabase.
  2. Admin edits to Neo4j — When you change a restaurant's cuisine/city/tags in admin, sync it to Neo4j so recommendations stay accurate.
  3. Tab navigation history — PWA back-swipe on tabs goes to Google sign-in. Need pushState for tab changes.
  4. Normalize all IDs to strings — Mixed number/string IDs cause silent comparison failures.
Medium Priority
  1. DM/Notifications/Filter → pushState — PWA back-swipe should close modals.
  2. Notes & lists → Supabase — Currently localStorage only. Lost on device change.
  3. Chat history → Supabase — Same issue. Local only.
  4. Chinatown city group — Listed in 6 city groups. Could cause cross-city filtering bugs.
Future
  1. Merge → update user_data.loved — Old loves still reference deleted restaurant IDs.
  2. Watchlist + Ratings → Neo4j — Would improve recommendation accuracy.
  3. Real-time sync — Push updates to other logged-in users when data changes.
